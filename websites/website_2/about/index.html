Purpose

Why this website exists

This fixture resembles a small production website while exposing routes that make server behavior easy to inspect. Each page explains which protocol or configuration feature it exercises.

Expected request flow

Your server should accept the connection, parse the request line and headers, select the correct server and location block, validate the method and body size, resolve the resource, and construct a valid HTTP response.
